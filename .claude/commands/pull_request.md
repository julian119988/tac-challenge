# Generate Pull Request

Create a comprehensive PR title and description.

## Variables
context_json: $1

## Instructions

- Parse the `context_json` which contains: issue_number, issue_title, plan_file
- Generate a PR following this format:

**Title:**
`<type>: <concise description> (fixes #<issue_number>)`

**Body:**
```markdown
## Summary
<1-3 bullet points summarizing changes>

## Changes
<bullet list of key changes made>

## Testing
<how to test these changes>

## Related Issue
Fixes #<issue_number>

```

- Link to the issue using "Fixes #<number>" to auto-close on merge
- Be specific about what changed and why
- Include testing instructions
- Keep it concise but informative

## Example

**Title:**
feat: add user authentication with OAuth (fixes #42)

**Body:**
## Summary
- Implemented OAuth 2.0 authentication flow
- Added login/logout functionality
- Integrated with Google and GitHub providers

## Changes
- Created authentication service in `src/auth/`
- Added OAuth provider configurations
- Updated user model with OAuth fields
- Added login/logout UI components

## Testing
1. Start the application
2. Click "Login with Google" or "Login with GitHub"
3. Complete OAuth flow
4. Verify user is authenticated
5. Test logout functionality
