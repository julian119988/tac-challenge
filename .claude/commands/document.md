# Generate Documentation

Create or update documentation for the changes made.

## Instructions

- Review the changes using git diff
- Identify what documentation needs to be created or updated:
  - Code comments and docstrings
  - README updates
  - API documentation
  - Architecture docs
  - User guides
- Generate appropriate documentation:
  - Follow existing documentation style
  - Be clear and concise
  - Include examples where helpful
  - Update existing docs if they're now outdated

## Documentation Types

1. **Code Comments**: Explain complex logic inline
2. **Docstrings**: Document functions, classes, modules
3. **README**: Update setup, usage, or feature descriptions
4. **API Docs**: Document endpoints, parameters, responses
5. **Architecture**: Explain design decisions and patterns

## Format

- Use markdown for documentation files
- Follow language-specific conventions for docstrings
- Keep it concise but comprehensive
- Include examples for non-obvious functionality

## Documentation

Review the changes and generate appropriate documentation.
