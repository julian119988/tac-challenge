# Code Review

Perform a comprehensive code review of the changes.

## Instructions

- Review all changes in the worktree using git diff
- Analyze code for:
  - **Correctness**: Logic errors, edge cases, potential bugs
  - **Best Practices**: Code style, patterns, conventions
  - **Security**: Vulnerabilities, injection risks, authentication issues
  - **Performance**: Inefficient algorithms, unnecessary computations
  - **Maintainability**: Code clarity, documentation, naming
  - **Testing**: Test coverage, test quality

## Review Format

```markdown
# Code Review

## Summary
<Overall assessment of the changes>

## Strengths
- <positive aspect 1>
- <positive aspect 2>

## Issues Found

### Critical
- <critical issue 1>
- <critical issue 2>

### Moderate
- <moderate issue 1>
- <moderate issue 2>

### Minor
- <minor issue 1>
- <minor issue 2>

## Recommendations
1. <recommendation 1>
2. <recommendation 2>

## Approval Status
[APPROVED / CHANGES REQUESTED / NEEDS DISCUSSION]
```

## Review

Perform the code review and provide structured feedback.
